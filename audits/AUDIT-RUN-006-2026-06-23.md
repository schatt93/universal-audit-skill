## 3. Findings

### F01 — Title version label is stale (`v4`) — **S3 (Medium)**
- **Location:** Line 1 (H1) vs Line 3 (metadata).
- **Evidence:** H1 reads `# Universal Audit, Verification & Test — Master Skill (v4)`. The metadata line directly below reads `Version 7.1 · 2026-06-21 · Supersedes v7.`, and the filename is `…-v7.1.md`. The title is three+ major versions behind.
- **Why it matters:** The title is the single most visible identifier. A reader or any tool keying on the H1 would mis-version the document — a traceability defect in an artifact that otherwise maintains strict version/run discipline.
- **Impact:** Low–Med · **Likelihood:** High (anyone reading the title) · **Status:** Open · **Resolved by:** R01.

### F02 — Cross-reference convention undercounts single-letter modules ("A–W" should be "A–Z") — **S3 (Medium)**
- **Location:** Line 13 (Cross-reference convention); recurs in the v7 changelog (~line 386).
- **Evidence:** The convention states modules are "a one-letter code (`A`–`W`) or two-letter code (`AA`–`AP`…)." But Modules **X** (Integrity), **Y** (Metadata), and **Z** (Image) are single-letter codes (Group 2). Programmatic check: single-letter modules actually span **A–Z** (26 of them); two-letter span AA–AP. So the one-letter bound is wrong by three modules. The group-range labels elsewhere ("Group 2 — Modules X–AE") are correct; only this letter-count description is defective.
- **Why it matters:** This sits in the *normative* convention that tells an executor how to parse module references. Stating one-letter codes stop at W contradicts the document's own X/Y/Z modules.
- **Impact:** Low–Med · **Likelihood:** Med · **Status:** Open · **Resolved by:** R02.

### F03 — "Both modes for each universal pass" contradicts Pass B4 and the §6 gate — **S3 (Medium)**
- **Location:** Lines 62 and 109 and 322 (blanket "each") vs Line 125 (B4) vs Line 294 (§6 gate).
- **Evidence:** The Stage B header (109) says "run **BOTH** the constructive and the adversarial mode **of each**"; the stage-flow (62) and §9 exit criteria (322) likewise say *each* universal pass runs in both modes. But **Pass B4** (125) is the "Cross-Cutting Adversarial / Red-Team" pass — inherently single-mode (adversarial only). The §6 dual-mode quality gate (294) correctly scopes the requirement to "**Pass B1–B3**." So the blanket statements (62/109/322) are internally contradicted by B4's design and by §6.
- **Why it matters:** An executor following §9/Stage-B literally would either invent a nonexistent "constructive B4" or judge a correct run as incomplete. §6 already contains the correct scoping, so the document disagrees with itself.
- **Impact:** Med · **Likelihood:** Med · **Status:** Open · **Resolved by:** R03.

### F04 — Finding-record schema (§5) has no field for the CVSS / OWASP data §4 mandates — **S4 (Low)**
- **Location:** Line 264 (§4) vs Line 285 (§5 finding-record fields).
- **Evidence:** §4 states "Security findings additionally carry a **CVSS** score; AI findings reference the relevant **OWASP LLM/Agentic** category." The §5 `Finding record fields` list is: `Finding ID · Stage/Module · Location · Description · Evidence · Category · Severity · Impact · Likelihood · Status · Tags · Resolved by`. There is **no** CVSS field and no OWASP-category field, so mandated data has no defined home.
- **Why it matters:** A schema that can't hold data the taxonomy requires forces ad-hoc placement (Evidence/Category), undermining the structured output the document prizes.
- **Impact:** Low · **Likelihood:** Med (only security/AI findings) · **Status:** Open · **Resolved by:** R04.

### F05 — Self-referential noun drift: "Skill" vs "instruction set" vs "prompt" — **S5 (Info)**
- **Location:** Line 1 ("Master **Skill**"), Line 7 ("**instruction set**"), Lines 13 & 70 etc. ("this **prompt**").
- **Evidence:** The document names itself three ways. The cross-reference convention even hardcodes one: "a bare `§N` in this **prompt**…" while the title calls it a Skill and the filename calls it a skill.
- **Why it matters:** Minor, but it's exactly the "synonym creep" the document's own Pass B3 flags. Pick one primary noun for self-reference.
- **Impact:** Negligible · **Likelihood:** n/a · **Status:** Open · **Resolved by:** R05.

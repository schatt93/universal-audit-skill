## 3. Audit of the Audit
- **The editor lied; git told the truth.** F010-1/2 were invisible to the Read view and only surfaced by checking the **shipping artifact** (`git show HEAD:file`) and unambiguous `grep`/`tail` on the mount. Lesson reinforced (Principle 9): verify against ground truth — the artifact that actually ships — not a possibly-stale view.
- **False-positive discipline paid off.** My own parity tooling produced false readings *three times* (a regex that can't match all-caps module names like `API`/`SEO`, a digit-after-code case `AC 3D/CAD`, a grep `[LICENSE]` bracket-expression, a case mismatch on `Fresh`). Per the directive "do not accept failure of a task as the result," each failing/odd measurement was re-derived until it agreed with ground truth — which is exactly what exposed the two real truncations while discarding the false alarms.
- **Severity calibration:** the truncations are S2 (shipped artifacts materially broken), not S1 (no safety/data-loss in the consuming systems). Date is S3 (audit-trail integrity).
- **Coverage:** structure/namespace/schema (CI 13/13), every Write-tool file's tail verified, lite↔master parity 42/42, platform formats (validated in AUDIT-RUN-009, unchanged). Standards currency carried from AUDIT-RUN-008.

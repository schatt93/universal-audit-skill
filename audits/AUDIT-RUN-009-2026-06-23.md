## 2. Findings

### F09-1 — Methodology lacked ground-truth / precondition validation (from live-test feedback) — **S2 (High)** · Resolved
- **Trigger:** Live ralph-loop run on a real project. A plan dated 2026-06-22 assumed a starting state (a UI package did not exist); the actual repo (2026-06-24 handoff) already had that package built with passing tests. The downstream agent noticed the drift only while implementing and treated it as a silent "reconcile and fill gaps," **not a finding** — so the audit/loop never flagged that executing the stale plan risked **regressing working code**.
- **Root cause:** asymmetry — Principle 1 validates *external* facts rigorously, but nothing mandated validating the artifact's claims about its **own project's current state** (what exists/builds/passes). A stale precondition therefore passed B1/B3 unflagged.
- **Resolved by R09-1:** new **Principle 9 (ground-truth reconciliation)** + hooks in **Pass B1** (verify claims about the system's current state against the actual system), **Stage A baseline** (verify stated assumptions/baseline vs reality), and a **§9 edge case** ("Stale precondition / spec-vs-reality drift" → finding, severity scaled by regression/rework/data-loss risk; in loop use, surface as a finding, never absorb into implementation). Mirrored into the Lite edition. Status **Resolved** (v8.0).
